## 什么是嵌入式？

嵌入式系统，简单说就是跑在专用硬件里的计算机系统。不像 PC 能装各种软件，嵌入式设备通常只做一件特定的事，但要做到极致——低功耗、高实时性、高可靠性。

常见的嵌入式设备：
- 消费电子：智能手表、无人机、路由器
- 汽车电子：ECU、车载娱乐、ADAS
- 工业控制：PLC、伺服驱动、传感器
- IoT：智能家居、可穿戴设备

嵌入式工程师的工作就是：写代码让这些硬件活起来——从底层驱动到上层应用，从裸机程序到 RTOS/Linux。

嵌入式工程师大致可以分为两类：硬件和软件，根据自己的兴趣选择对应的篇章阅读。

## 软件篇

### C 语言是基本功

嵌入式 90% 的代码是 C，别只停留在考试水平，至少要能：
- 指针、结构体、位运算烂熟于心
- 看懂反汇编，理解函数调用栈

这玩意熟能生巧，多写就行了。

### 单片机入门（STM32）

从 51 过渡到 STM32 是标准路线（主播就是）。

强烈推荐江协科技的教程：
- [C51 教程](https://www.bilibili.com/video/BV1Mb411e7re/?spm_id_from=333.337.search-card.all.click&vd_source=ab428c3952dca788271ad95654273af5)
- [STM32 教程](https://www.bilibili.com/video/BV1th411z7sn/?spm_id_from=333.337.search-card.all.click)

他的影响太大了，身边所有学习单片机的全部看过他的视频，他将是你大学最好的老师（当然本人恩师赵虚左）。

学习要点：
- GPIO、中断、定时器、PWM、ADC → 外设驱动
- I2C / SPI / UART → 通信协议
- DMA + 中断 → 高效数据搬运

重点不是调库，是看懂数据手册和原理图。

开发环境推荐：上面学完可以试试 CUBEMX + VSCODE（B 站搜）。大人，时代变了！AI 的介入让嵌入式开发更加简单。当然建议基础学完了再 vibecoding，不然一直点 Enter 也学不到啥。

### 你的第一个项目

学完上面这些大概要一个学期，这时候你就可以动手做你第一个项目了。我这边推荐几个：
- 32 循迹小车（中）
- 四旋翼无人机（难）
- 智能曼波小狗（简）
- 智能手表（简）

B 站一般都有教程，直接搜就行了，这种东西都烂大街了，还有千万别写简历里，会被笑的。

做完真的很有成就感，如果代码是自己手写的，那恭喜你，已经超过 80% 的人了，你对单片机的理解将会更上一层楼。

推荐网址：[OSHWHub](https://oshwhub.com/)

### RTOS

裸机跑通了就上 RTOS（推荐 FreeRTOS）：
- 任务调度、优先级抢占
- 信号量、消息队列、事件组
- 中断与任务的同步

可以试试 ESP32 了，国产芯片。推荐教程：[ESP32 教程](https://www.bilibili.com/video/BV1mhWqzFEzn/?spm_id_from=333.337.search-card.all.click&vd_source=ab428c3952dca788271ad95654273af5)

我必须提醒的是：嵌入式学习没有终点，没有任何一个人敢说对单片机了如指掌，这只是冰山一角。

### Linux 嵌入式方向

其实本科生到这里就可以止步了，下面是研究生该深入的。当然你觉得你是人中龙凤，或者年薪想 30w+，冲 Linux 吧，少年！

Linux 方向薪资上限更高，技术栈也更深：
- 应用层：交叉编译、文件 I/O、多线程、网络编程
- 驱动层：字符设备驱动、设备树、platform 总线
- 内核裁剪：Buildroot / Yocto 构建系统

推荐教程：[Linux 嵌入式教程](https://www.bilibili.com/video/BV1DJ4m1M77z/?spm_id_from=333.1387.favlist.content.click&vd_source=ab428c3952dca788271ad95654273af5)
